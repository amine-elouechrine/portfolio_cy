Analyseur Lexical MOCA

Programme C d'analyse lexicale avec refactoring complet, optimisation algorithmique et 0 leak mémoire

Durée:3 mois
Organisation:Ensimag
Rôle:Développeur C
C Valgrind GDB Makefile CUnit Testing Algorithm Optimization

Le Défi

Transformer une application monolithique C présentant des défauts structurels et des problèmes de gestion mémoire en un système robuste et optimisé respectant les meilleures pratiques de développement.

Le projet consistait à développer un analyseur lexical générant un dictionnaire d'occurrences de mots avec tracking précis des positions (ligne, colonne), tout en suivant une approche d'amélioration continue.

  • Refactoriser du code monolithique en architecture modulaire
  • Créer des bibliothèques statiques et dynamiques réutilisables
  • Éliminer toutes les fuites mémoire (validation Valgrind)
  • Optimiser les performances (×3 d'amélioration)
  • Implémenter une suite de tests complète (95% de couverture)

Mon Rôle

Développeur C principal, responsable de l'ensemble du cycle de refactoring et d'optimisation :

  • Analyse de code : audit du code existant, identification des défauts structurels
  • Refactoring : redesign modulaire, séparation des responsabilités
  • Développement de bibliothèques : création de libs statiques/dynamiques avec APIs claires
  • Gestion mémoire : utilisation de Valgrind et AddressSanitizer pour détecter et corriger les leaks
  • Tests et validation : suite de tests CUnit, fuzzing automatisé
  • Optimisation : profiling avec gprof, amélioration des algorithmes
